# Hospital Management System - Service Layer Practice

## Objective

Build Repository, Service, DTO, and Controller layers for the Hospital Management System.

Do NOT use Exception Handling, Security, JWT, Pagination, or Specifications.

Use Spring Data JPA's built-in methods wherever possible.

---

# Rules

1. Controllers must never access repositories directly.

2. Controllers talk only to Services.

3. Services talk to Repositories.

4. DTOs should be returned from Controllers.

5. Entities should never be exposed directly from Controllers.

6. Use constructor injection.

7. No field injection.

8. No exception handling for now.

9. No ModelMapper. Perform mapping manually.

10. Build Service Interfaces before Service Implementations.
